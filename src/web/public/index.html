fusion-agent

AI-powered developer toolkit — vibe coder, live debugger, autonomous agent, and session manager in one.

⚡
Vibe Coder
Chat with AI, auto-write files to disk. Interactive or autonomous mode.
🤖
Autonomous Agent
Give it a requirements file and walk away. It loops until done.
🔍
Live Debugger
Tail logs, batch with AI, deduplicate errors, push to Jira / GitHub.
🔗
Integrations
Git fixes pushed to GitHub API. Jira tickets. Copilot issue assignment.

Quick Start

# Install globally
npm install -g fusion-agent

# Set at least one provider key
export OPENAI_API_KEY=sk-...

# Interactive vibe coder
ai-agent chat

# Live debugger on a Docker container
ai-agent debug --docker my-api --ui

# Web dashboard
ai-agent ui

Getting Started

Requirements: Node.js 18+, npm 9+, and an API key for OpenAI, Anthropic, or Gemini.

Installation

# Global CLI
npm install -g fusion-agent

# Project dependency
npm install fusion-agent

# From source
git clone https://github.com/fury-r/fusion-agent.git
cd fusion-agent && npm install && npm run build

Configuration

Set environment variables or create .fusion-agent.json in your project root.

# Environment variables
export OPENAI_API_KEY=sk-...
export ANTHROPIC_API_KEY=sk-ant-...
export GEMINI_API_KEY=AIza...
// .fusion-agent.json
{
  "provider": "openai",
  "model": "gpt-4o",
  "port": 3000,
  "github": {
    "token": "ghp_...",
    "repoUrl": "https://github.com/org/repo"
  }
}

Config search order

  1. .fusion-agent.json (current directory)
  2. .fusion-agent.yaml (current directory)
  3. ~/.fusion-agent/config.json
  4. ~/.fusion-agent/config.yaml

CLI flags and env vars always override the config file.

CLI config command

ai-agent config --provider openai --model gpt-4o
ai-agent config --show   # print resolved config

⚡ Vibe Coder

An AI pair-programmer that reads your project, generates code, and writes files to disk automatically. Two modes: Interactive chat and Autonomous.

How file generation works

AI response
→
Detect ```lang:path``` blocks
→
Guardrail check
→
Write to disk
→
Record in session

The AI must produce code blocks in this format:

```typescript:src/middleware/auth.ts
// full corrected file content
```

CLI

ai-agent chat [options]

  -p, --provider <name>     openai | anthropic | gemini
  -m, --model <name>        e.g. gpt-4o
  -s, --session <name>      create or resume a named session
  -k, --speckit <name>      agent persona (default: vibe-coder)
  -g, --guardrail <rule>    add a guardrail rule (repeatable)
      --context             inject project dir structure

In-session commands

Command Action
/exit End session and save
/save Save without ending
/turns Show conversation history
/context Inject project directory tree

Web UI usage

  1. Navigate to ⚡ Vibe Coder in the sidebar.
  2. Select Chat or Autonomous tab.
  3. Type a message and press Enter or click Send.
  4. Watch files appear in the Files Changed panel in real time.

Programmatic API

import { AgentCLI } from 'fusion-agent';

const agent = new AgentCLI({ provider: 'openai' });
const session = agent.createSession({
  name: 'my-project',
  speckit: 'vibe-coder',
  projectDir: process.cwd(),
});

const turn = await session.chat('Add JWT auth middleware', {
  stream: true,
  onChunk: (chunk) => process.stdout.write(chunk),
});
console.log('Files changed:', turn.fileChanges);

🤖 Autonomous Agent

Reads a requirements file (or inline text), implements it step-by-step, detects loops, and asks for human guidance via a Human-in-the-Loop (HIL) modal when stuck.

How it works

Read requirements
→
AI generates plan
→
Implement step N
→
Write files
→
Loop detect?
→
REQUIREMENTS_COMPLETE

Web UI

  1. Go to ⚡ Vibe Coder → 🤖 Autonomous tab.
  2. Enter a requirements file path or paste requirements text.
  3. Optionally add rules (one per line) and set a time/step limit.
  4. Click ▶ Run. Progress streams live.
  5. If a HIL event fires, a modal appears — type guidance and click Continue →.

Config reference

interface AutonomousConfig {
  requirementsFile?:         string;   // path to .md or .txt
  requirementsContent?:      string;   // inline text
  rules?:                    Rule[];   // constraints per step
  timeLimitSeconds?:         number;   // 0 = no limit
  maxSteps?:                 number;   // default 50
  stuckThreshold?:           number;   // default 3
  loopWindowSize?:           number;   // default 4
  loopSimilarityThreshold?:  number;   // default 0.85
}

Events

Event Payload Description
step StepResult Step completed
complete StepResult[] All steps done
hil HILRequest Needs human guidance
error Error Fatal error

🔍 Live Debugger

Tails log sources (files, Docker, processes, HTTP), batches lines, sends them to AI for analysis. Deduplicates repeated errors. Results stream to the Web UI in real time via Socket.IO.

Flow

Log source
→
Filter (level/pattern)
→
Line buffer
→
Deduplicate
→
AI analysis
→
Web UI + Actions

CLI

ai-agent debug [options]

  # Source (one required)
  -f, --file <path>           watch a log file
  -d, --docker <container>    attach to Docker stdout/stderr
  -c, --cmd <command>         spawn a process

  # Filtering
  --log-level <levels>        e.g. ERROR,FATAL
  --log-pattern <patterns>    comma-separated regex

  # Batching
  --batch <n>                 lines per AI call (default 20)
  --retry <n>                 max retries (default 3)

  # Notifications
  --notify-slack <url>        Slack webhook
  --notify-teams <url>        Teams webhook

  # Web UI
  --ui                        launch dashboard
  --port <port>               dashboard port (default 3000)

Error deduplication

The same error fingerprint is never sent to the AI twice. Repeated occurrences show an amber badge on the analysis card and increment a counter. No extra AI cost.

Dashboard tab

The Dashboard tab inside a debugger session shows: unique error count, suppressed repeat count, total events, and a bar chart of the top errors by repeat count.

Actions per analysis

Button Action
Create Jira Issue Files a Jira ticket with the AI analysis as description
Apply Git Fix Commits AI-proposed code changes directly to GitHub via API
Assign to Copilot Creates a GitHub issue and assigns the Copilot coding agent

Apply Git Fix — 3-step wizard

  1. Configure — enter Token, Remote URL, Branch, PR Title.
  2. Preview Changes — side-by-side diff of every file the AI wants to modify.
  3. Apply — commits directly to GitHub via the Git Data API, no local git needed.
Provide a GitHub token and Remote URL to use the GitHub API path. Repo Path is only needed for the local-git fallback.

Socket.IO events

Event Direction Payload
debugger:analysis Server → Client { sessionId, analysis, meta }
debugger:log Server → Client { sessionId, line }
debugger:error-repeated Server → Client { turnId, repeatCount, lastSeen }
debugger:subscribe Client → Server { sessionId }

💾 Session Manager

Sessions persist conversation history (turns), file changes, and debugger metadata to disk. They can be resumed, exported, and deleted.

REST API

Method Path Description
GET /api/sessions List all sessions
GET /api/sessions/:id Get session detail + turns
POST /api/sessions Create a new session
DELETE /api/sessions/:id Delete one session
DELETE /api/sessions Delete all sessions
GET /api/sessions/:id/export Export session as JSON

Session turn object

{
  id:               string,
  timestamp:        string,      // ISO
  userMessage:      string,
  assistantMessage: string,
  fileChanges?:     FileChange[],
  usage?:           TokenUsage,
  debuggerMeta?:    DebuggerTurnMeta
}

Web UI

  1. Click Sessions in the sidebar to list all sessions.
  2. Click a session row to view its turns and metadata.
  3. Use ↻ Refresh to reload, 🗑 Clear All to wipe all sessions.
  4. Use ⬇ Export on the detail page to download the full session JSON.

🔵
Google Gemini
gemini-1.5-pro, gemini-1.5-flash, gemini-2.0-flash

Streaming

All providers support streaming. Pass stream: true and onChunk to session.chat() to receive tokens as they arrive.

Token limits & auto-truncation

The live debugger auto-detects token limits from 429 responses and truncates future prompts automatically. Set --log-token-limit <n> to override.

🔗 Integrations

Git Fix (GitHub API)

Commits AI-proposed code changes directly to GitHub without a local clone using the Git Data API.

Field Required Description
Token ✓ GitHub PAT with repo scope
Remote URL ✓ https://github.com/owner/repo
Branch — Target branch (default: fusion-agent/auto-fix)
Base Branch — Base for PR (default: main)
PR Title — If set, a pull request is opened automatically
Repo Path — Local path — only needed when token is absent

Jira

Creates a Jira issue from a live debugger analysis turn.

// POST /api/debugger/:sessionId/jira
{
  jiraConfig: {
    baseUrl:    "https://your-org.atlassian.net",
    email:      "you@example.com",
    apiToken:   "...",
    projectKey: "ENG",
    issueType:  "Bug"
  },
  turnId: "turn-abc123",
  summary: "Optional custom title"
}

GitHub Copilot Agent

Creates a GitHub issue and assigns the Copilot coding agent via agent_assignment.

// In .fusion-agent.json
{
  "github": {
    "token": "ghp_...",
    "repoUrl": "https://github.com/org/repo",
    "autoAssignCopilot": true
  }
}
When autoAssignCopilot is enabled, the Apply Git Fix action is blocked to avoid conflicting PRs.

🛡 Guardrails

Guardrails are safety rules that gate file writes, git commits, and Copilot issue creation.

File / Git guardrail rules

Rule format Effect
allow-path:src/ Only files under src/ may be modified
deny-path:secrets/ Files under secrets/ are never touched
max-files:5 At most 5 files per commit
Use TypeScript strict mode Custom rule injected as AI constraint

Copilot issue guardrails

Rule format Effect
deny-keyword:password Block if title or body contains the word
require-label:bug Issue must include the bug label
max-title-length:120 Issue title must be ≤ 120 chars
max-body-length:5000 Issue body must be ≤ 5000 chars

Adding guardrails

# CLI
ai-agent chat --guardrail "deny-path:secrets/" --guardrail "max-files:5"

# Programmatic
const session = agent.createSession({
  guardrails: [createGuardrail('custom', 'Use TypeScript strict mode')]
});

🎭 Speckits

Speckits are prebuilt agent personas — a name, description, system prompt, and example prompts. They are set when creating a session and shape the AI's behaviour for the entire session.

Built-in speckits

Name Best for
vibe-coder General coding, file generation, refactoring
debugger Log analysis, root cause, code fixes
code-review PR review, security, performance feedback
cluster-debugger Kubernetes / multi-service remediation

Using a speckit

# CLI
ai-agent chat --speckit debugger

# Programmatic
agent.createSession({ speckit: 'code-review' });

Custom speckits

import { registerSpeckit } from 'fusion-agent';

registerSpeckit({
  name: 'security-auditor',
  description: 'Reviews code for OWASP Top 10 vulnerabilities',
  systemPrompt: `You are a security expert...`,
  examples: ['Review this Express route for SQL injection'],
});

    🎫 Create Jira Ticket

    ⚙ Apply Git Fix

    1 Configure › 2 Review Changes › 3 Apply
    Loading changes…

    🤖 Assign to GitHub Copilot

    Creates a GitHub issue and assigns it to the Copilot coding agent, which will autonomously open a fix PR.

    Rules: deny-keyword:<word> · require-label:<label> · max-title-length:<n> · max-body-length:<n>

    ⚠ Copilot Assignment Blocked

    You can override this guardrail and assign to Copilot anyway, or dismiss to skip this analysis.
